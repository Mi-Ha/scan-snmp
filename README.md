

#Запуск приложения

Перед запуском приложения следует:
<li> Создать базу данных PostgreSQL с именем "scan_snmp"
<li> Настроить подключение к БД в файле dev.data.application.properties (модуль data, папка resources).
<li> Залить бэкап из файла scan_snmp.backup (если СУБД - PostgreSQL), либо запустить на выполнение два файла (модуль data, пакет example.group.sd.data.run): сначала DropCreateDatabaseScheme, затем FillSettings.

С исходными настройками сканер сканирует только адрес 127.0.0.1.

#Настройки сканнера
Настроечные файлы находятся в папке resources модуля data:

<li> Настройки подключения к БД - файл dev.data.application.properties
<li> Настройки сканера  - файл scanner.properties.json
<li> Настройки протокола SNMP - snmp.properties.json

###Настроечные параметры сканера:
- ip - адрес сервера, на котором работает сканер
- networkPrefixLength - ранг подсети, на основе которой вычисляется пул адресов для сканирования (используется только если не задан список адресов listIp, см. ниже). Допустимые значения: от 0 до 31.  Для значения 0 диапазон будет максимальный (0.0.0.1-255.255.255.254), для значения 31 - минимальный.
- netScanPeriod -	пауза между циклами сканирования, мс
- waitingForPollingDevices - максимальное время отклика от устройств, мс
- listIp":["127.0.0.1"] - список IP адресов, которые будут опрашиваться по SNMP протоколу. Если список не задан, то адреса будут вычислены в соответствии с параметром networkPrefixLength.

###Настроечные параметры протокола SNMP:
- smnpPort - порт для опроса по SNMP протоколу
- smnpTransportProtokol - протокол ("udp", "tcp")
- smnpCOMMUNITY - наименование группы
- smnpRETRIES - число попыток опроса одного устройства
- smnpTIMEOUT - время ожидания ответа от устройства, мс
- smnpVersion - версия SNMP

После корректровки настроечных файлов следует перезалить их в базу данных (запустить файл FillSettings)
